# Git Commit Skill

## Description

自动化 Git 代码提交流程，将当前工作区的所有修改内容合并为单次提交，并推送到远程仓库。遵循项目 Git 提交规范，确保提交消息清晰、准确且符合中文最佳实践。

## Use Cases

- 完成功能开发后提交代码
- 修复 bug 后提交代码
- 重构代码后提交代码
- 任何需要将代码变更持久化到 Git 仓库的场景

## Core Rules

1. **单次提交**：所有修改必须合并为一次提交，禁止拆分为多次提交
2. **工作区干净**：提交完成后工作区必须干净，不允许残留未提交的修改
3. **中文消息**：提交消息必须使用中文，按照最佳实践总结修改内容
4. **自动推送**：提交成功后自动执行一次推送远程仓库操作

## Usage

当用户表达以下意图时，自动触发此 skill：
- "提交代码"
- "commit 代码"
- "提交修改"
- "推送代码"
- "git commit"
- 其他表达需要提交代码到 git 仓库的类似表述
